# HIBIKI UI/UX計画書 v1.0

**作成日:** 2026-06-14

---

## 現状分析

開発者都合のメニュー構造から、ユーザー（親分・子分）の行動密度に基づいたUIへの刷新を検討している。

---

## 検討案

### A案：フロー特化型（業務フェーズ駆動）

業務のフェーズ（受注 → 配車 → 実行 → 請求 等）に沿って画面を構成する方式。ユーザーが今やるべき作業に沿って自然に遷移できるUIを目指す。

### B案：階層型Finder駆動（コマンドパレットUI）

Finder的な階層ナビゲーションとコマンドパレットを組み合わせ、任意の機能へ素早くアクセスできる方式。キーボード・検索による高速操作を重視する。

### C案：ラベル多重所属（タグベースUI）

エンティティ（案件・ドライバー・車両等）に複数ラベルを付与し、ラベル軸で横断的に参照・フィルタリングする方式。クロス参照と柔軟なグルーピングを実現する。

---

## 考慮事項

- **親分アプリの全体像把握** — カレンダー表示の可変性を活かし、俯瞰と詳細の切り替えを容易にする。
- **子分アプリのスマホ入力最適化** — フォントサイズ・タップ領域・入力フロー等、現場での片手操作を前提とした設計。
- **階層が深くなる問題の解消** — クイックアクセス・お気に入り機能の活用により、よく使う機能への到達コストを下げる。
- **クロス参照（多重所属）と検索ファインダーの有用性** — タグベースの横断参照と、Finder型検索の組み合わせが業務効率に与える効果を評価する。
